# Playlist Generation Flow

This documents the full pipeline triggered by `POST /api/generate`, implemented in `backend/controllers/generation.controller.ts` (~1150 lines).

## High-Level Flow

```
POST /api/generate
  │
  ├─ 1. Validate request + auth + rate limit
  ├─ 2. Load user's liked songs from DB
  ├─ 3. Parse vibe → emotion targets
  ├─ 4. Build genre profile from library
  ├─ 5. Score every track against targets
  ├─ 6. Select and sequence tracks
  ├─ 7. Create Spotify playlist + add tracks
  ├─ 8. Save to DB (saved_playlists + playlist_history)
  └─ 9. Return response
```

## Phase 1 — Validation

1. Check `req.session.spotifyUserId` → 401 if missing.
2. Check Spotify feature flag → 503 if not configured.
3. **Rate limit**: check in-memory `Map<userId, number[]>` of recent request timestamps. If ≥5 requests in the last 60 seconds → 429.
4. Validate request body with Zod (`GeneratePlaylistRequest`):
   - `vibe`: string, 1–140 chars.
   - `mode`: `"strict" | "balanced" | "chaotic"` (default `"balanced"`).
   - `length`: integer 10–100 (default 40).
   - `referencePlaylist`: optional Spotify playlist URL.

## Phase 2 — Load Library

1. Query `liked_songs` WHERE `spotify_user_id = userId`.
2. If no rows → 400 (`"Your library is empty or not yet synced"`).
3. Rows are loaded into memory for scoring. For large libraries (thousands of tracks), this is a full table scan per request.

## Phase 3 — Parse Vibe → Emotion Targets

The vibe string is analysed to extract:
- `energy`: float 0–1 (e.g., "night drive" → low, "gym workout" → high)
- `valence`: float 0–1 (positivity)
- `tempo`: BPM target range
- `acousticness`: float 0–1
- `danceability`: float 0–1
- `timeOfDay`: "morning" | "afternoon" | "evening" | "night" | null
- `environment`: "indoor" | "outdoor" | "vehicle" | null
- `nostalgia`: float 0–1
- `genre_hints`: array of genre family strings

The parsing uses keyword matching and heuristic rules applied to the vibe string. If `KWALIFY_DETERMINISTIC=1`, the parsing uses fixed seeds.

If a `referencePlaylist` URL is provided:
1. Fetch the playlist's tracks from Spotify's API (using user tokens).
2. Average the audio features of those tracks.
3. Use those averages to bias the emotion targets.

## Phase 4 — Build Genre Profile

1. Check in-memory `genreProfileCache` (warmed after each sync).
2. If cache miss, read all `liked_songs` rows and compute genre distribution from artist names and album names using keyword matching against a set of ~18 genre family patterns.
3. The genre profile is a `Map<genreFamily, count>` representing what fraction of the library belongs to each genre family.

This step helps the scoring engine weight tracks from genres that the vibe implies more highly.

## Phase 5 — Track Scoring

For each track in the library:

```
score = 0
score += energy_weight    × (1 - |track.energy - target.energy|)
score += valence_weight   × (1 - |track.valence - target.valence|)
score += tempo_weight     × tempo_proximity(track.tempo, target.tempo_range)
score += danceability_weight × (1 - |track.danceability - target.danceability|)
score += acousticness_weight × (1 - |track.acousticness - target.acousticness|)
score += genre_weight     × genre_match(track, genre_profile, vibe_genre_hints)
score += nostalgia_weight × nostalgia_score(track.addedAt, target.nostalgia)
```

Weights vary by `mode`:
- **Strict**: high weights on energy + valence, low weights on genre and nostalgia. Tightly matches the vibe.
- **Balanced**: moderate weights across all dimensions, includes artist diversity boost.
- **Chaotic**: lower primary weights, higher secondary dimension weights, adds random jitter to scores.

Tracks with NULL audio features (no Spotify data) fall back to defaults (`energy: 0.5, valence: 0.5, tempo: 120`, etc.) and receive a small penalty.

## Phase 6 — Track Selection

After scoring:
1. Sort tracks by score descending.
2. Apply diversity constraints in `balanced` mode:
   - Max 3 tracks per artist in the final selection.
   - Attempt to include tracks from at least 3 different decades if available.
3. Take the top `length` tracks (after diversity filtering).
4. Sequence the final tracklist: ordered by a smooth energy arc (gradual build or gradual descent based on the vibe's `timeOfDay`).

## Phase 7 — Create Spotify Playlist

1. Call `getValidAccessToken()` to ensure a fresh token.
2. Call Spotify `POST /v1/users/{userId}/playlists`:
   - Name: `"Kwalify: {vibe}"` (truncated to 100 chars).
   - Description: `"Generated by Kwalify — {vibe}"`.
   - Public: `false`.
3. Add tracks in batches of 100 (Spotify API limit) using `POST /v1/playlists/{playlistId}/tracks`.
4. Returns `{spotifyPlaylistId, spotifyPlaylistUrl}`.

## Phase 8 — Persist to Database

Two writes happen:

**`saved_playlists`** — the canonical playlist record:
```json
{
  "userId": "...",
  "name": "Kwalify: ...",
  "emotionProfile": { energy, valence, timeOfDay, environment, nostalgia, ... },
  "tracks": [{ trackId, trackName, artistName, albumName, albumArt }],
  "spotifyUrl": "https://...",
  "vibe": "...",
  "mode": "balanced"
}
```

**`playlist_history`** — the generation log:
```json
{
  "spotifyUserId": "...",
  "playlistId": "spotify_playlist_id_string",
  "playlistUrl": "https://...",
  "name": "...",
  "vibe": "...",
  "mode": "...",
  "trackCount": 40,
  "emotionProfile": {...},
  "trackIds": ["spotify_track_id", ...]
}
```

## Phase 9 — Response

```json
{
  "playlistName": "Kwalify: late night drive on empty roads",
  "spotifyPlaylistUrl": "https://open.spotify.com/playlist/...",
  "trackCount": 40,
  "tracks": [...],
  "playlistId": 17,
  "emotionProfile": { "energy": 0.45, "valence": 0.3, "timeOfDay": "night", ... }
}
```

`playlistId` is the integer `saved_playlists.id` (used to build share URLs `/p/:id`).

## Status Tracking

During generation, a status map `Map<userId, string>` tracks the current phase. This is read by `GET /api/generate/status`. Phases:
- `"idle"` (default)
- `"loading_library"`
- `"parsing_vibe"`
- `"building_profile"`
- `"scoring"`
- `"selecting_tracks"`
- `"creating_playlist"`
- `"saving"`
- `"done"`

The frontend does not currently poll this endpoint.

## Error Scenarios

| Scenario | Response |
|---|---|
| Not authenticated | 401 |
| Spotify not configured | 503 |
| Rate limit exceeded | 429 |
| Library empty / not synced | 400 |
| Spotify API failure (token expired, network) | 500 with error message |
| Zero tracks scored above threshold | 400 (`"No matching tracks found"`) |
| Playlist creation fails on Spotify | 500 |
