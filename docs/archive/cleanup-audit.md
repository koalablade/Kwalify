## Notes On Unused Imports And Exports

- No high-confidence unused imports were identified by text-reference search in active runtime files.
- A stricter unused-symbol pass should be reviewed separately before code deletion. A read-only secondary audit reported compiler-confirmed unused locals/imports across several files when using `tsc --noEmit --noUnusedLocals --noUnusedParameters`; those findings need per-symbol triage because some may be compatibility/debug scaffolding.
- `tsconfig.json` currently does not enable `noUnusedLocals` or `noUnusedParameters`, so unused import/helper detection is not compiler-enforced.
- Recommended future audit step: run a dedicated TypeScript unused-export tool or temporarily enable `noUnusedLocals` in a throwaway branch. Do not enable it directly on `main` without triaging expected noise from type-only compatibility exports.
