# PRIMARY ISSUE

The system appears capable of detecting moods/scenes correctly.

However:

* semantic interpretation is not strongly influencing final outputs
* genre ecosystems are not being enforced
* personalized mode appears to dilute requested vibes
* no-library mode is slightly better but still drifts
* requests such as:

"dirt road with a dog"

"outlaw country"

"petrol station at 2am"

"rainy city lights"

still generate many unrelated tracks

This indicates a retrieval/ranking/routing problem rather than a keyword problem.

# PHASE 3 — RETRIEVAL BEFORE SCORING

→ retrieve everything
→ score later

→ filter retrieval pool
→ retrieve ecosystem candidates
→ score candidates
→ build playlist

Scene interpretation must affect retrieval,
not only final ranking.
