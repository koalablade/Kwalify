# PROJECT RESTRUCTURE & CODEBASE CLEANUP

You are a senior software engineer performing a codebase cleanup.

Your goal is to make the project easier for a new developer to understand within 10 minutes.

## OBJECTIVE

Refactor the file structure, naming, and organization for clarity.

Do NOT change application behaviour.

Do NOT modify backend business logic.

Do NOT introduce new features.

Focus only on:

* file names
* folder names
* component names
* function names
* variable names
* project organization
* dead code removal

---

## REMOVE

Delete:

* unused files
* duplicate files
* abandoned experiments
* legacy code
* commented-out code blocks
* unused components
* unused assets
* unused hooks
* unused utilities
* unused stylesheets
* unused routes

If a file is not imported and not required:

Delete it.

---

## RENAME FOR HUMANS

Avoid names like:

* temp
* test2
* newVersion
* finalFinal
* generatorV2
* engineX
* manager
* handler
* processor
* util
* helper

Replace with names that describe exactly what the file does.

Examples:

playlistGenerator.ts
instead of
generator.ts

spotifyImport.ts
instead of
handler.ts

playlistResults.tsx
instead of
ResultsManager.tsx

userPreferences.ts
instead of
prefsEngine.ts

---

## FOLDER STRUCTURE

Organize by feature, not by technical type.

Bad:

/components
/hooks
/utils
/pages

Good:

/features/playlist-generation
/features/spotify-import
/features/results
/features/settings

Each feature should contain its own:

* components
* hooks
* services
* types

---

## FILE SIZE RULES

Split files that exceed:

* 300 lines for components
* 500 lines for services

Large files should be broken into clearly named modules.

---

## IMPORT CLEANUP

* Remove unused imports
* Remove circular dependencies
* Simplify import paths
* Consolidate duplicate utilities

---

## NAMING RULES

Names should answer:

"What does this actually do?"

Good:

generatePlaylist.ts
saveUserSettings.ts
spotifyPlaylistFetcher.ts

Bad:

engine.ts
manager.ts
core.ts
system.ts
controller.ts

---

## DOCUMENTATION

Add concise comments only where necessary.

At the top of major files add:

Purpose:
Responsibilities:
Dependencies:

Avoid obvious comments.

---

## OUTPUT REQUIRED

After restructuring provide:

1. New folder tree
2. Files removed
3. Files renamed
4. Duplicate code consolidated
5. Dead code removed
6. Any architectural issues discovered

Do not stop after analysis.

Perform the cleanup and apply the changes.
