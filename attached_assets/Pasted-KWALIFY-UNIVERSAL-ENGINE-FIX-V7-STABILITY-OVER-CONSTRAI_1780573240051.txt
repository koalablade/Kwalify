KWALIFY UNIVERSAL ENGINE FIX (V7 — STABILITY OVER CONSTRAINTS)

You are a senior music recommendation systems engineer.

Your job is to FIX the playlist engine so it produces:

natural, Spotify-like playlists
consistent genre diversity
accurate scene/mood matching
no genre collapse
no over-filtering

This must work across ALL genres, eras, moods, and scenes.

⚠️ CORE PROBLEM

The system is currently over-constrained:

Scene detection is too dominant
Ecosystem lock behaves like a hard filter
Pre-filter removes too much diversity
Genre weighting is too weak or incorrectly applied
Final output collapses into a single genre (e.g. country-only)
🎯 PRIMARY GOAL

Replace rigid filtering with probabilistic ecosystem balancing.

The system must behave like a DJ, not a classifier.

🧠 KEY FIX PRINCIPLE

❌ DO NOT hard-filter genres based on scene
❌ DO NOT collapse pools aggressively before scoring
❌ DO NOT enforce single-genre dominance

✔ DO use scene as a soft directional signal
✔ DO use genre as a distribution system
✔ DO preserve cross-genre adjacency
✔ DO prioritise listening experience over strict correctness

🧩 CRITICAL ARCHITECTURE CHANGE
1. Replace ecosystem lock (hard filter → soft distribution)

Current system:

scene → allowed genres only → scoring

FIX:

Scene must output a GENRE WEIGHT MAP, not a whitelist.

Example:

“outlaw country / dirt road sunset” →

{
  country: 0.40,
  americana: 0.20,
  southern_rock: 0.15,
  folk: 0.10,
  blues: 0.10,
  rock: 0.05
}

NO GENRE IS BLOCKED — only weighted.

2. Remove hard pre-filter collapse

Pre-filter must NOT:

remove full genres
remove entire ecosystem groups
reduce pool below diversity threshold

Instead:

✔ Only remove:

corrupted tracks
invalid metadata
explicit blacklisted content

Everything else stays in scoring pool.

3. Replace “ecosystem lock” with “genre pressure scoring”

Instead of filtering:

Modify scoring:

finalScore =
  semantic * 0.35 +
  emotion * 0.20 +
  sceneMatch * 0.15 +
  aesthetic * 0.10 +
  library * 0.10 +
  genreAffinity * 0.10–0.20

Where:

genreAffinity =

+boost if matches scene distribution map
-slight penalty if outside map
NEVER zeroed out
4. Add GENRE DIVERSITY ENFORCER (post-score step)

After scoring:

Ensure:

max 40% of playlist from one genre
minimum 3 distinct genres per playlist
no genre monoculture allowed

If violated:
→ re-balance selection, NOT re-filter pool

5. STOP “POOL CAPPING THAT DESTROYS DIVERSITY”

Current issue:

pool reduced too early
diversity lost before scoring

Fix:

keep full candidate pool longer
only apply mild trimming based on metadata quality
final selection should come from a large pool
6. SCENE SYSTEM FIX

Scene detection must:

Return:

primary scene
confidence
secondary scene candidates (top 2)

Rules:

If confidence < 0.65:
→ blend scenes (do not lock)
If confidence ≥ 0.85:
→ strong weighting only, NOT filtering
7. REMOVE OVER-AGGRESSIVE FALLBACKS

Never do:

“if pool too small → reopen full library”

Instead:

expand via adjacent genres only
loosen weights slightly
keep ecosystem coherence intact
8. DEBUG SYSTEM REQUIREMENTS

Update debug output to ALWAYS show:

Scene
detected scene
confidence
secondary scenes
Genre system
genre weight map (from scene)
pre-score genre distribution
post-score genre distribution
final playlist genre breakdown
Filtering
pool size before scoring
pool size after scoring
% removed (metadata-only filtering only)
Diversity
genre entropy score
max genre dominance %
diversity compliance (PASS/FAIL)

Every playlist must feel like:

✔ coherent mood

✔ no “single genre lock-in”
✔ emotionally consistent flow

NOT:

90% one genre

rigid ecosystem locking

DO:

DON’T:
